Embedded Content

  • <embed>: embed external content. e.g. external app, browser plug-in, or other source of interactive content
  • <fencedframe>: similar to <iframe>, but with more native privacy features. Represents nested browsing context
  • <iframe>: Represents nexted browsing context, embedding another HTML page into the current one
  • <object>: Represents an external resource, which can be treated as an image, a nested browsing context, or a resource to be handled by a plug-in
  • <picture>: give the browser a set of display options for image content. Contains zero or more <source> elements and one <img> element. Browser will consider each source and choose the best, else if no matches will use the img 'src' attribute.
  • <portal>: don't use.
  • <source>: use with <picture>, <audio>, or <video>. it's a void element (self-closing). commonly used to offer the same media content in multiple formats.

What is a Browser Plug-in?

Any external module that runs in the browsers. They are kind of OUTDATED, or PHASED OUT (e.g. Flash) due to security and compatibility issues. EXTENSIONS are the more common term now.

Some potentially useful Browser Plug-ins:

  • Lighthouse: Google Chrome plug-in to optimize websites, audit performance, SEO, accessibility, and best practices.
  • Web Developer Toolbar: Powerful set of web dev tools
  • PageSpeed Insights
  • Web Vitals
  • PerfectPixel: Overlay tool when working from design mockups (e.g. Figma)
  • Netlify Dev Tools: after deploying with Netlify, monitor traffic, uptime, and deploy previews
  • SEOquake
  • Grammarly
  • BrowserStack
