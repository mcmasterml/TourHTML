HTML Inline Text Semantics

Define the meaning, structure, or style of text

  • <a>: hyperlink, use with attribute <href>
  • <abbr>: abbreviation or acronym
  • <b>: Don't use (basically). Draw reader's attention to content. DON'T use for styling or granting importance. use CSS <font-weight> instead for styling, and <strong> instead for granting importance
  • <bdi>: isolate and handle bi-directional scripts (e.g. hebrew)
  • <bdo>: override text directionality. render in different direction.
  • <br>: self-closing, line break.
  • <cite>: citation
  • <code>: computer code
  • <data>: think enum. links a given piece of content with a machine-readable translation. associate text with a value (passed as attribute e.g. "value=100" to <data>). use <time> for time/date-related data
  • <dfn>: the term being defined. the term is defined by the ancestor <p>, the <dt>/<dd> pairing, or nearest ancestor of the <dfn> element. see Links to definitions
  • <em>: emphasis
  • <i>: italics, designates a range set off from the normal text
  • <kbd>: textual user input
  • <mark>: highlighted or marked text, denotes reference or notation purposes due to relevance in enclosing context
  • <q>: inline quotation. compare with <blockquote>, the non-inline relative
  • <ruby>: ruby is textual annotation, typically used to show the pronunciation of East Asian typography. renders small annotations above, below, or next to base text.
  • <rp>: ruby annotation to provide fall-back parenthesis (not all browsers support ruby notation)
  • <rt>: ruby annotation to provide pronunciation, translation, or transliteration
  • <s>: strikethrough. However, for indicating document edits, use <del> and <ins> instead
  • <samp>: output from a computer program. typically rendered in monospace
  • <small>: side-comments, copyright and legal text. independent of its styled presentation
  • <span>: generic inline container for phrasing content. the inline version of <div>
  • <strong>: indicates importance, seriousness, or urgency
  • <sub>: subscript
  • <sup>: superscript
  • <time>: time. include datetime attribute to translate dates into machine-readable format (for better SEO or custom features such as reminders)
  • <u>: underline, probably don't use
  • <var>: represents variable name in a math or programming context
  • <wbr>: optional line break, use for mobile styling? if you want a hyphen, use the soft-hyphen character entity &shy;

Use <kbd> to signify user input
