Forms

  • <button>: issa button. takes input (e.g. mouse, keyboard, finger, voice command, etc.), performs an action (e.g. submit form)
  • <datalist>: e.g. search list. allows anything to be input, but also can contain a set of <option> elements guiding user choice
  • <fieldset>: groups several controls as well as <label>s within a web form
  • <form>: section containing interactive controls for submitting information
  • <input>: POWERFUL! defines interactive controls for web-based forms to accept user data. see all types and attributes
  • <label>: represents a caption for an item in a user interface
  • <legend>: child of <fieldset>, representing a caption for the content
  • <meter>: represents a scalar value within a known range, or a fractional value (e.g. gas tank, scalar graph)
  • <optgroup>: e.g. sections in a dropdown list. creates groupings of options within a <select>
  • <option>: child of either <optgroup> or <datalist>, defining the items contained therein. can represent menu items etc.
  • <output>: container element, returning results of a calculation or result of user action
  • <progress>: Indicator showing completion progress
  • <select>: a control that provides a menu of options
  • <textarea>: think longtext. represents a multi-line, plain-text editing control, for sizeable amounts of free-form text (e.g. comment, review, feedback)
